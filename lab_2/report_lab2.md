После создания базовой конфигурации был добавлен новый виртуальный диск. Далее он был включён в RAID-массив, после чего расширено пространство LVM и увеличена файловая система. В результате логический том `lv0` стал больше.

### Состояние перед добавлением нового диска
Сначала было зафиксировано текущее состояние системы до расширения массива.

### Добавляем новый виртуальный диск
Новый диск был подключён средствами VMware. После этого система определила новое устройство, которое можно было использовать для расширения массива.

![alt text](img/image-22.png)
![alt text](img/image-23.png)
![alt text](img/image-24.png)

![alt text](img/image-25.png)

### Очищаем диск
Перед добавлением диск был очищен от возможных сигнатур файловых систем и старых метаданных.

```bash
sudo wipefs -a /dev/sde
```

### Добавляем в RAID
Добавляем новый диск в уже существующий массив.

```bash
sudo mdadm --add /dev/md0 /dev/sde
```

### Увеличиваем RAID
После добавления устройства массив был переведён на использование четырёх дисков.

```bash
sudo mdadm --grow /dev/md0 --raid-devices=4
```

![alt text](img/image-26.png)

### Расширяем LVM
После увеличения RAID-массива система LVM также должна узнать о новом свободном пространстве.

#### Обновляем PV
Команда перечитывает размер физического тома LVM.

```bash
sudo pvresize /dev/md0
```

#### Расширяем LV
Всё свободное место группы томов добавляется к логическому тому `lv0`.

```bash
sudo lvextend -l +100%FREE /dev/shamenkov/lv0
```

#### Расширяем файловую систему
После увеличения логического тома расширяется и сама файловая система.

```bash
sudo resize2fs /dev/shamenkov/lv0
```

![alt text](img/image-27.png)


```bash
./easyrsa gen-req server nopass
./easyrsa sign-req server server
```

### Сгенерировали сертификаты на клиенте

```bash
./easyrsa gen-req client1 nopass
./easyrsa sign-req client client1
```

### Генерация параметров Диффи—Хеллмана

```bash
./easyrsa gen-dh
```

### Копирование файлов на сервер

```bash
sudo cp pki/ca.crt /etc/openvpn/
sudo cp pki/private/server.key /etc/openvpn/
sudo cp pki/issued/server.crt /etc/openvpn/
sudo cp pki/dh.pem /etc/openvpn/
```

### Настройка конфигурационного файла `server.conf`

## Задание 21 — Настроить проброс портов 139 и 445 из сети 172.23.0.0/24 в VPN-сеть 10.8.0.0/24

Заключительный этап лабораторной работы состоял в настройке проброса портов SMB в VPN-сеть. Для этого были добавлены правила DNAT и разрешена пересылка пакетов, а также включён NAT для обратного прохождения трафика.

### Делаем проброс портов

### Проброс входящих соединений
Входящие подключения на порты `139` и `445` перенаправляются на узел в VPN-сети.

```bash
sudo iptables -t nat -A PREROUTING -i ens33 -p tcp --dport 139 -j DNAT --to-destination 10.8.0.2:139
sudo iptables -t nat -A PREROUTING -i ens33 -p tcp --dport 445 -j DNAT --to-destination 10.8.0.2:445
```

### Разрешаем пересылку
Разрешаем прохождение соответствующего трафика через сервер.

```bash
sudo iptables -A FORWARD -p tcp -d 10.8.0.2 --dport 139 -j ACCEPT
sudo iptables -A FORWARD -p tcp -d 10.8.0.2 --dport 445 -j ACCEPT
```

### Включаем NAT
Для корректного обратного маршрута был добавлен `MASQUERADE`.

```bash
sudo iptables -t nat -A POSTROUTING -o tun0 -j MASQUERADE
```
